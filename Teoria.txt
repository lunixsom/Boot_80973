        PROMESAS EN JS ( 1:15:30 hs a  )


            Cuando trabajamos con Promesas lo mas interesante es ir a la pestaña de "NETWORK" de la herramienta del desarrollador de Chrome.

            En la PROMESAS nosotros vamos a quedar a la espera de que esa promesa se resuelva satisfactoriamente o de manera negativa, osea que el servidor nos responde o no pudo obtener la data porque se demoro demasiado por que el servidor esta caido, o porque la peticion tiene un error o lo que pediste el servidor no sabe resolverlo.

            Pero yo a traves de las PROMESAS PUEDO ESTABLECER UN CONTRATO ENTRE MI CODIGO DE JS Y EL SERVIDOR y entender que cuando ese contrato se cumpla significa que el servidor ya me respondio INDEPENDIENTEMENTE DEL TIEMPO QUE HAYA PASADO, sea nada casi o sea un monton lo que yo tenga que esperar.


            Otro tarea que se puede considerar asincrona, en la cual nosostros no sabemos cuando va a suceder, es el LISTENER, JS no sabe cuando va a suceder esa tarea por eso es que lo trata asincronamente para continuar con las demas tareas que si puede resolver.


            QUE ES UNA PROMESA ? (1:24:30 hs)

                Por lo general no se llega a crear una promesa manualmente/artesanalmente (a mano). pero ahi va el ejemplo:

                    let respuesta

                    const promesa = new Promise((resolve, reject) => {
                        setTimeout(() => {
                            resolve("La promesa se ha cumplido")  || o tambien puede responder reject(1000)
                        }, 2000)
                    })


                    promesa
                        .then((valorResuelto) => {
                            console.log("Promesa resuelta")
                        })
                        .catch((error) => {
                            console.warn("hubo un error en la promesa")        
                        })

                    El metodo THEN se va ejecutar cuando la promesa se resuelva, a su vez la promesa tambien tiene la posibilidad de ejecutar un metodo catch() que es cuando la promesa se rechaze.
                    Entonces yo tengo dos posibilidades o dos caminos a tomar. 

                    En el catch cuando yo atrapo el error de la promesa, siempre voy a recibir un parametro que es el error.

 ASYNC/AWAIT ( 19 min a )   

    IMPORTANTE: La palabra reservada "async" se le aplica a la funcion padre de "await"

    Usar then y catch esta bueno pero hoy en dia hay una solucion un poco mas moderna para esperar que una promesa se    resuelva.

    Tambien para evitar los promise hell o callback hell se usa un metodo mas nuevo y amigable: ASYNC/AWAIT

    Lo negativo de async await es que piero la posibilidad de detectar algun error para esto lo que se suele combinar con una funcion asincrona es la estructura/bloque TRY CATCH (que ojo no tiene siempre que usarse con peticiones, es una estructura nativa de JS)


    TRY CATCH ( 58 min a 1:15 hs)

        TRY CATCH con una funcion asincrona es una combinacion para lograr una mejor fluidez o un mejor manejo de mi codigo.

        Lo interesante de esta estructura es que me atrapa el error que pueda acontecer en una peticion o en cualquier error que este adentro de la estructura y permite que JS siga ejecutandose sin tener que pincharse.

        Cuando algo falla dentro de mi bloque TRY se dispara mi bloque CATCH que atrapa el error y la ejecucion del flujo continua y no se pincha, osea no me bloquea la ejecucion de mi codigo.
